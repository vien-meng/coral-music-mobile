# B5-21 空列表本地音频导入

- 阶段：Batch 5 / Phase 4
- 状态：DOING
- 负责人：Codex
- 开始时间：2026-07-21

## 根因与目标

“导入本地音频”仅存在于已打开播放列表的工具栏。新安装或没有播放列表时，用户无法打开系统音频选择器。

在“我的列表”页直接提供文件导入入口，复用现有扫描和 SQLite 写入链；有效音频首次导入时创建并打开“本地音乐”列表，不创建空列表。

## 实施与验证

- 空列表和已有列表的“我的列表”页工具栏均提供音频文件选择入口；文件扫描成功后创建或复用“本地音乐”，导入后自动打开该列表。
- 系统分享仍写入既有“分享导入”列表；共享导入与手动导入共用同一个控制器方法，但各自指定目标列表，不复制扫描或去重逻辑。
- 无有效音频时直接反馈 0 项，不创建“本地音乐”空列表。
- `dart format`、定向 `dart analyze` 与 `git diff --check` 通过。2026-07-21 已在 SM-N986U 构建、安装并启动 Debug APK，无启动崩溃；文件选择器交互待在设备上手动点击验证，任务保持 `DOING`。
