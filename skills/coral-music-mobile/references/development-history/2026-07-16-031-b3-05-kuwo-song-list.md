## 目标、范围、不做内容与依赖

实现酷我热门歌单广场、歌单详情、详情歌曲点击和播放全部，形成从在线发现到现有播放器的完整路径。

依赖 B1-05、B2-05、B4-11。不实现标签分类、排序、多源歌单、搜索歌单、收藏、导入、分页追加或离线缓存；先验证桌面端同一酷我公开接口在移动端的最小切片。

## 桌面端基线与确认行为

对照 `coral-music-desktop/src/renderer-react/services/musicSdk/sdk/kw/songList.js` 的 `getList`、`getListDetail`、`filterList` 和 `filterListDetail`，以及 `SongListRoutePanel.tsx`。桌面端打开广场后加载默认热门歌单，打开详情后可播放全部或从歌曲点击开始；移动端采用原生页面切换，不复制桌面双栏。

## 实施方案、关键接口、数据与平台差异

- 共享领域新增 `OnlinePlaylist` 与 `PlaylistDetail`；仅保存显示和播放实际需要的数据。
- 独立 `KuwoPlaylistService` 复用现有 Dio、错误映射、文本/音质解析，使用 HTTPS 端点；不扩张通用在线目录接口以避免未接入来源出现空实现。
- 状态控制器保存广场与当前详情，采用请求序号隔离旧响应；详情点歌/播放全部复用队列和 `PlayerController`。
- 不涉及平台桥接、存储和 User API 协议变化；播放仍受已启用音源支持限制。

## 验收、风险与恢复入口

- 收口时补固定桌面响应 fixture 契约测试、状态旧响应隔离、歌单详情点歌和播放全部 Widget Test；Android 真机验证真实 HTTPS 列表加载。
- 风险为第三方公开端点变动；出现端点不兼容时记录响应并按既有来源可行性制度处理，不静默降级为假数据。

关联 P2-04、P2-07、功能矩阵“歌单”“队列”；后续为标签/排序、多来源和收藏导入。

- 详情支持播放全部和点歌，统一进入内存队列与播放器；第三方端点真机和 fixture 契约待本批集中验证。
