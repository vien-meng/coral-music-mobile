# B5-13 在线歌单本地收藏快照

- 阶段：Batch 5 / Phase 4
- 状态：DOING
- 负责人：Codex
- 开始时间：2026-07-20
- 完成时间：未完成

## 目标与范围

用户可在在线歌单详情收藏或取消收藏。设备本地保存来源、稳定歌单 ID、名称、封面、作者、描述和当前曲目快照；在“我的收藏”可离线打开、播放或移除。

不做：远端订阅同步、跨设备同步、服务器存储、歌单下载或自动刷新远端内容。

## 实施方案

- 在现有 `LibraryStore` SQLite 数据库新增一张快照表和 v4 显式迁移；曲目复用现有安全的 `Track` JSON 快照格式。
- 收藏按钮和已有歌曲收藏按钮共用 `LibraryController` 的本地状态刷新机制；不增加网络请求、Repository 或依赖。
- 快照作为收藏当下的固定内容，远端歌单变更不覆盖本地内容。

## 依赖与验收

- 依赖 B5-01 SQLite、B3-05 在线歌单详情和 B5-12 本地收藏基础。
- 收藏后关闭网络仍能在“我的收藏”播放；取消收藏只删除对应快照，不影响歌曲收藏或用户列表。

## 实际修改与验证

- `LibraryStore` 升级到 SQLite schema v4，新增 `online_playlist_favorite`；主键是 `source:playlistId`，保存歌单元信息和当时曲目 JSON 快照。
- `LibraryController` 增加收藏歌单查询、切换和独立修订号；不使用网络、账号或服务器。
- 歌单详情新增收藏按钮；“我的收藏”将收藏歌单与收藏歌曲分区展示，点击收藏歌单用快照替换队列并播放第一首，右侧按钮可移除快照。
- `flutter analyze` 通过；设备离线播放与迁移真机验收待下一次 Android 安装，状态保持 `DOING`。
