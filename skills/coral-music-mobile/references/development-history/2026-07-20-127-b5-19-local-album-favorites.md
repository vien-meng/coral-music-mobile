# B5-19 本地专辑收藏快照

- 阶段：Batch 5 / Phase 4
- 状态：DOING
- 负责人：Codex
- 开始时间：2026-07-20
- 完成时间：未完成

## 目标与范围

允许用户在已有专辑分类详情收藏/取消收藏整张专辑；本机保存专辑标题、代表封面和当时曲目快照，在“我的收藏”可离线播放或移除，并进入本机备份。

不做：远端专辑订阅、账号同步、云端数据、自动刷新远端曲目或单独建设专辑详情 API。

## 方案与依赖

- 复用 `Track` 快照、SQLite 迁移、收藏页和分类详情，新增最小 `FavoriteAlbum` 领域值与一张本地快照表。
- 稳定 key 优先使用首曲来源/专辑 ID；无 ID 时退化为来源、专辑、歌手组合，避免同名不同来源互相覆盖。
- 备份沿用 v1 容器增加可选字段，旧备份解码保持兼容。

## 验收

- 收藏、取消、离线播放、重启读取、备份编解码和 SQLite migration；不影响歌曲、歌单收藏或本地列表。

关联：`B5-14`、`B5-13`、`B6-14`、`P4-06`。

## 实际修改与验证

- `FavoriteAlbum` 以本地稳定 key、专辑名、歌手、封面和曲目快照表示收藏；SQLite 升级 v11，新增 `album_favorite`，创建和升级路径均显式迁移。
- 专辑分类的详情面板仅在“专辑”页签展示收藏/取消按钮；收藏页新增“收藏专辑”分区，可离线替换队列播放或删除，不误将艺术家、类型、年份分组作为专辑收藏。
- 相同曲目快照也进入分类索引；本机备份 v1 增加可选 `favoriteAlbums` 字段，旧备份未包含该字段仍可安全解码和恢复。
- 新增备份专辑快照回归断言；`dart format` 与 `git diff --check` 已通过。Flutter 静态检查、迁移测试、APK 和真机回归继续受当前 SDK 执行授权用量限制阻断。
