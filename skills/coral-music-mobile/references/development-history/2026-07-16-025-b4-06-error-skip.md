## 目标、范围、不做内容与依赖

当当前曲目的解析、加载或播放发生错误时，在本轮队列中自动尝试下一首未失败歌曲；所有可选歌曲均失败后保留明确错误状态，绝不无限循环重试。

依赖 B4-01、B4-03、B4-04、B4-05。不实现换源、音质降级、网络指数退避、失败持久化或错误上报；它们需要来源/缓存和诊断能力后另行实施。

## 桌面端基线与确认行为

对照桌面端 `coral-music-desktop/src/renderer-react/stores/domains/playerStore.ts` 的统一队列切换路径。桌面端播放失败不能让列表播放永远停在同一失效项；移动端沿用既有队列选歌和 `playTrack()` 入口，不在 UI 侧重复错误策略。

## 实施方案、关键接口、数据与平台差异

- `PlayerController` 在当前曲目失败时记录本轮失败的稳定 `Track.id`，然后按队列当前模式取得下一项。
- 只自动尝试尚未失败的不同歌曲；单曲或全队列均失败则保留 `AppFailure`，以避免循环请求失效 URL。
- 用户手动选择歌曲时允许重新尝试该项；不改平台音频、User API 和数据库。

## 验收、风险与恢复入口

- 收口时覆盖“第一首失败跳至第二首”“两首均失败后停止”“手动重试已失败歌曲”三类单元测试，并在 Android 真机用受控失效 URL 加有效 URL 回归。
- 当前恢复入口为 `lib/features/player/state/player_controller.dart`；实际改动和验证将在任务结束时补充。

关联 P3-03、功能矩阵“生命周期”“队列”；后续为音质降级/换源和播放进度持久化。

- 无可用候选时保留明确错误状态，手动点歌会移除该曲失败标记并允许重新尝试。受控失效 URL 真机回归待后续集中执行。
