# B4-05 播放模式与随机历史

- 阶段：Batch 4 / Phase 3

- 负责人：Codex

- 完成时间：2026-07-17

## 目标、范围、不做内容与依赖

实现列表循环、单曲循环、随机三种播放模式，并在随机自动切歌时记录本轮已经播放的队列索引，优先选择未播放曲目以避免短时间重复。播放详情提供可见且可操作的模式切换入口。

依赖 B4-03 和 B4-04。不实现“顺序播放至末尾停止”、错误跳过、队列编辑/持久化、后台媒体会话或跨设备同步；这些均不应借播放模式任务混入。

## 桌面端基线与确认行为

对照 `coral-music-desktop/src/renderer-react/stores/domains/playerStore.ts` 的 `getQueueSibling`、`getHistoryFilteredCandidateIndexes` 与 `clearPlayedHistory`。桌面端自动切歌按模式选择候选：列表循环首尾衔接，单曲循环保留当前项，随机优先本轮未播放歌曲；手动上一首/下一首在这三种模式下按列表循环移动。

## 实施方案、关键接口、数据与平台差异

- 在共享领域层增加 `PlaybackMode`，队列状态保存当前模式。
- `PlaybackQueueController.selectAfterCompletion()` 统一实现三种自动切歌策略；`selectNext`/`selectPrevious` 继续保持手动顺序循环。
- 随机候选以队列索引集合记录；队列重置、模式切换或一轮耗尽时清理并重新记录当前项。随机源允许测试注入，不新增依赖或持久化数据。
- 播放详情只读取/写入队列状态；音频、User API 与平台桥接不分叉。

## 验收、风险与恢复入口

- 批次收口时验证三种模式、随机一轮不重复、手动切歌与自动完成事件；Android 真机在设备恢复后补连续播放回归。
- 实际修改、设计调整和验证结果将在本任务完成时补充；恢复入口为 `lib/domain/music.dart`、`lib/features/player/state/playback_queue_controller.dart`、`lib/features/player/state/player_controller.dart` 与播放详情页。

关联 P3-03、功能矩阵“播放模式”“队列”“生命周期”；后续为错误跳过和播放进度持久化。
