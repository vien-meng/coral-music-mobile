# UI-18 首页顶部菜单

- 阶段：高保真 UI / 在线发现
- 状态：DOING
- 负责人：Codex
- 开始时间：2026-07-20

## 目标与范围

修复首页铃铛按钮无响应，并优化音乐平台选择菜单。铃铛点击后显示锚定消息面板，当前没有消息数据源时显示“暂无新消息”；音乐平台选择继续复用现有榜单来源状态和切换逻辑，只替换默认下拉框的表现。

本任务不新增通知持久化、推送或网络接口。

## 验证要求

- 点击铃铛显示“消息通知”和“暂无新消息”。
- 点击音乐平台按钮显示带当前选中状态的平台列表。
- 选择平台后菜单关闭，并沿现有控制器加载该平台榜单。

## 实施与验证

- 首页顶部栏使用 Material 3 `MenuAnchor` 替换默认平台下拉框；菜单统一使用主题表面色、8dp 圆角和轻描边，平台项显示图标及当前平台勾选状态。
- 铃铛按钮使用相同锚定菜单显示“消息通知”和“暂无新消息”，不创建虚假的消息数据。
- 平台选择继续调用既有 `LeaderboardController.selectSource()`，未增加新的来源状态或请求路径。
- `test/app_shell_test.dart` 更新来源切换操作，并增加消息空态回归检查。
- 针对性 `dart analyze` 与 `git diff --check` 通过。
- `flutter test test/app_shell_test.dart` 因工作区外 Flutter SDK 缓存锁需要审批；审批服务返回 HTTP 503，测试未启动，因此任务保持 `DOING`。
- Samsung SM-N986U / Android 13 已完成本轮 Debug APK 构建、安装和首帧启动；启动阶段保存音源请求返回 HTTP 200，真实 FLAC 进入 ExoPlayer 解码并持续播放，未出现 Flutter 红屏或进程崩溃。
- ADB 自动点击、界面树和截图读取仍因外部审批服务返回 HTTP 503 未执行，首页两个菜单尚缺自动化真机截图证据。
