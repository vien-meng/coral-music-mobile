# 本地优先、无服务器能力范围修订

- 阶段：计划修订 / 全项目
- 状态：DONE
- 负责人：Codex
- 开始时间：2026-07-17
- 完成时间：2026-07-17

## 目标与范围

按产品决定，项目不提供评论、项目自有服务器存储、跨设备同步或局域网服务能力。

保留：在线音乐检索与播放、受限 User API、本地 SQLite、用户自行选择的本地文件、应用目录下载、用户自行配置的 WebDAV 文件源和本地文件备份。

移除：评论、列表跨设备同步、OpenAPI/局域网服务，以及与这些能力相关的发布和验收任务。

## 决策与影响

- WebDAV 不属于项目提供的服务器存储：应用仅直连用户自行配置的远程文件源，不提供账号、同步或数据托管。因此保留其产品入口、架构和后续计划。
- User API 仅用于受限访问第三方公开音乐来源，不上传或保存用户数据到项目服务器；脚本继续只在当前会话保存。
- 备份只保留本地文件导入/导出，不上传至项目或局域网服务器。

## 验收与后续

- `appDestinations` 保留“网盘资源”，但不提供项目自有的 WebDAV 账号服务或跨设备同步。
- `development-plan.md`、`feature-parity.md`、`architecture.md` 与 `SKILL.md` 不再把评论、跨设备同步或局域网服务列为范围或发布门槛。
- 本次不删除尚未使用的 `TrackSourceKind.webdav` 枚举：它没有产品入口或实现，后续若模型收敛需要移除时再做一次有编译回归的机械清理。

## 实施结果

- 已从范围、功能矩阵和架构中移除评论、跨设备同步和 OpenAPI/局域网服务。
- WebDAV 入口和任务已恢复并明确为“用户自行配置的远程文件源”；项目不提供 WebDAV 账号、用户数据托管或设备间同步。
- 已更新 `SKILL.md`、`development-plan.md`、`feature-parity.md`、`architecture.md` 和 `app_router.dart`；`git diff --check` 通过。
