## 目标与范围

复用已启用的受限 User API 音源，执行桌面协议的 `lyric` 动作，在播放详情显示 LRC 原文、翻译及当前行高亮。

不做：本地 LRC 优先、歌词数据库缓存、逐字 LX LRC、罗马音展示、编辑偏移及非 User API 在线歌词源。

## 桌面端对照

- `coral-music-desktop/src/renderer-react/services/musicSdkRuntime.ts` 对同一 User API 来源调用 `{ action: 'lyric', info: { musicInfo, isGetLyricx } }`，返回 `lyric`、`tlyric`、`rlyric`、`lxlyric`。
- 移动端只接受 JSON 对象或纯歌词字符串，限制响应大小；脚本仍只能使用既有 HTTPS 网络桥接。

## 实施方案与验收

- 原生 WebView 增加独立歌词请求回调，和播放 URL 回调隔离；manifest 同时记录 `musicUrl`、`lyric` 能力。
- Dart 解析标准 LRC 时间标签；UI 高亮当前行并在有对应时间线时显示翻译。
- 验收：未声明 lyric 的音源显示明确空态；合法歌词按播放位置变更高亮；脚本异常不影响当前播放。

## 当前进度

- Android `UserApiRunner` 已增加独立 `resolveLyric` 回调和超时/大小限制；`musicUrl`、歌词请求与清理互不复用 pending result，避免歌词失败影响正在取链。
- manifest 已同时返回 `musicUrlSources` 与 `lyricSources`；Dart 归一化桌面返回的裸对象、`data` 包装对象和纯字符串歌词。
- 已增加标准 LRC 时间轴解析，播放详情显示当前行高亮、同时间戳翻译并随当前行平滑跟随；未声明 lyric 能力或无有效时间轴时显示空态。
- 已新增 `test/lyric_timeline_test.dart`；聚焦 `flutter test`、`flutter analyze --no-fatal-infos`、Android Debug APK 构建通过。

## 2026-07-16 追加：LRC 时间顺序与自动滚动

- 播放详情歌词面板已在当前行变化时滚动到估算位置，避免用户在长歌词中丢失播放上下文；估算行高的限制以 `ponytail:` 注释标明，后续仅在真实排版明显漂移时改为逐项测量。
- 修正 `parseLyricTimeline`：原文 LRC 先按时间戳排序，再与翻译时间戳对齐。真实 User API 和本地 LRC 不能假设文本行已按时间顺序返回，否则当前行判定和自动滚动会错位。
- 验证：`flutter test test/app_shell_test.dart test/lyric_timeline_test.dart -r compact`、`flutter analyze --no-fatal-infos` 均通过；排序行为由 `orders LRC lines by timestamp instead of source line order` 覆盖。

## 2026-07-16 追加：罗马音

- 复用已有 `LyricPayload.rlyric`，按同一时间戳与原文 LRC 对齐，并在播放详情显示；不新增网络请求、设置项或缓存。
- `aligns LRC text and translation by timestamp` 同时覆盖翻译与罗马音的时间戳对齐。

## 2026-07-16 追加：LX 逐字歌词

- 对照桌面 `lyric-font-player`，仅在 `lxlyric` 含 `<start,duration>` 标签时优先使用它；词片段以歌曲绝对毫秒计算当前高亮，普通 LRC 保持原有行高亮回退。
- 验证由 `prefers LX LRC and retains word timing` 覆盖，确保普通歌词不会覆盖可用的逐字时间轴。
- 修正逐字片段不再 `trim` 非空文本，保留英文和混排歌词的词间空格；`preserves whitespace between timed LX words` 覆盖该行为。
- 支持桌面兼容的 `[offset:±ms]` 标签：原文行与 LX 词片段使用同一偏移，翻译/罗马音仍按其未偏移时间戳与原文对齐；`applies an LRC offset to lines and timed words` 覆盖。
