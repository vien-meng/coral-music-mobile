## 目标与范围

复用已启用的受限 User API 音源，执行桌面协议的 `lyric` 动作，在播放详情显示 LRC 原文、翻译及当前行高亮。

不做：本地 LRC 优先、歌词数据库缓存、逐字 LX LRC、罗马音展示、编辑偏移及非 User API 在线歌词源。

## 桌面端对照

- `coral-music-desktop/src/renderer-react/services/musicSdkRuntime.ts` 对同一 User API 来源调用 `{ action: 'lyric', info: { musicInfo, isGetLyricx } }`，返回 `lyric`、`tlyric`、`rlyric`、`lxlyric`。
- 移动端只接受 JSON 对象或纯歌词字符串，限制响应大小；脚本仍只能使用既有 HTTPS 网络桥接。

## 实施方案与验收

- 原生 WebView 增加独立歌词请求回调，和播放 URL 回调隔离；manifest 同时记录 `musicUrl`、`lyric` 能力。
- Dart 解析标准 LRC 时间标签；UI 高亮当前行并在有对应时间线时显示翻译。
- 验收：未声明 lyric 的音源显示明确空态；合法歌词按播放位置变更高亮；脚本异常不影响当前播放。

## 当前进度

- Android `UserApiRunner` 已增加独立 `resolveLyric` 回调和超时/大小限制；`musicUrl`、歌词请求与清理互不复用 pending result，避免歌词失败影响正在取链。
- manifest 已同时返回 `musicUrlSources` 与 `lyricSources`；Dart 归一化桌面返回的裸对象、`data` 包装对象和纯字符串歌词。
- 已增加标准 LRC 时间轴解析，播放详情显示当前行高亮、同时间戳翻译并随当前行平滑跟随；未声明 lyric 能力或无有效时间轴时显示空态。
- 已新增 `test/lyric_timeline_test.dart`；聚焦 `flutter test`、`flutter analyze --no-fatal-infos`、Android Debug APK 构建通过。
