# B3-23/B4-37 多来源回归修复

## 根因与目标

播放器从搜索页返回时保留了下层文本框焦点，系统恢复该焦点后弹出输入法。歌词链路只有酷我原生服务和公共 LRCLIB，QQ、网易云、咪咕、酷狗缺少桌面端已有的来源请求。网易云与酷狗搜索没有站点请求头，QQ/咪咕搜索封面字段覆盖不足。

退出播放器必须清除焦点；歌词按本地 -> 来源服务 -> LRCLIB 执行；搜索请求和封面解析对齐桌面端已使用的接口与字段。

## 验证要求

- 从播放器返回首页或搜索页不弹出输入法。
- QQ、网易云、咪咕、酷狗在线曲目都尝试来源歌词后再走独立兜底。
- 网易云、酷狗搜索携带对应站点请求头；QQ、咪咕搜索有可用封面字段时生成封面 URI。

## 实施与验证

- 播放详情页进入和销毁时调用 Flutter 焦点管理器清除已有文本焦点，避免下层搜索框在返回后恢复系统输入法。
- 新增独立 `SourceLyricService`：QQ 使用桌面端 `fcg_query_lyric_new` 协议，网易云使用歌曲歌词接口，咪咕使用响应携带的 LRC URL，酷狗按桌面端候选/下载 LRC 协议；任何来源失败仍继续 LRCLIB，不触碰 User API 取链运行时。
- 网易云搜索补 `Referer` 与 PC cookie，酷狗搜索补网页 `Referer`/`Origin`；QQ 解析优先使用响应直接封面地址，咪咕兼容 `imgItems` 和 `img1`–`img3`。
- `test/lyric_controller_test.dart` 覆盖来源歌词成功时不请求公共兜底；此前已存在的队列、音源和封面测试保持不变。
- `dart format`、定向 `dart analyze` 和 `git diff --check` 通过。Flutter 测试及真机回归待执行：当前设备上的 Release APK 尚未包含本轮代码。
