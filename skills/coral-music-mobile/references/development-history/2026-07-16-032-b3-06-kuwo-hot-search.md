## 目标、范围、不做内容与依赖

在搜索页尚未输入关键词时显示真实酷我热搜词；点击热词立即调用现有歌曲搜索流程。

依赖 B1-05、B3-01、B3-02。不实现本地搜索历史、跨来源热词、个性化推荐或服务端缓存；这些需要持久化和更多来源后再实现。

## 桌面端基线与确认行为

对照 `coral-music-desktop/src/renderer-react/services/musicSdk/sdk/kw/hotSearch.js`：调用 `hotword.kuwo.cn/hotword.s`，当响应 `status=ok` 时提取 `tagvalue[].key`。移动端使用 HTTPS 等价端点，不能沿用桌面 HTTP 明文请求。

## 实施方案、关键接口、数据与平台差异

- 独立 `KuwoHotSearchService` 复用现有 Dio 与错误映射；使用 `FutureProvider` 按页面首次需要加载并支持刷新。
- 搜索页将热词渲染为标准 Chip；点击回填输入框并调用已有 `SearchController.submit`，不复制搜索请求和分页状态。
- 无存储、平台通道或 User API 变化。

## 验收、风险与恢复入口

- 收口时补桌面响应 fixture 解析测试、加载失败重试和热词点击搜索 Widget Test；Android 真机确认 HTTPS 端点。
- 第三方端点若不支持 HTTPS，将记录可行性阻断，不能退回明文 HTTP。

关联 P2-03、搜索功能矩阵；后续为历史和综合搜索。
