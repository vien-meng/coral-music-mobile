## 目标、范围与依赖

实现设备内的在线歌曲下载：从已启用 User API 取链，写入应用文档目录，持久化任务并提供下载页。依赖 B4-01 取链和 B5-01 SQLite。

本切片不做系统后台任务、系统下载目录导出、并发调度或歌单下载全部；它们分别留给 B6 后续任务，不能伪造“后台下载已支持”。

## 技术决策

复用已安装的 Dio 与其传递依赖 `path_provider`，不新建网络客户端；URL 只在下载运行期间驻留内存，SQLite 仅保存曲目快照、所选音质、文件路径、状态和错误摘要。

## 当前实现

- 下载页、播放详情的在线歌曲下载入口、Dio 下载流和应用文档目录 `downloads/` 的原子 `.part` 写入已开始实现。
- 当前状态会写入 `LibraryStore` 的 SQLite schema v5；进程恢复后，原来的等待中或下载中任务会安全显示为“已暂停”，由用户明确继续，避免后台被系统中断后伪报完成。
- 已完成任务可从下载页本地播放；播放 URL、音源脚本和凭据均不写入数据库。

## 实际修改与验证

- 新增 `DownloadTask`/`DownloadStatus`、下载控制器和下载页；播放详情的在线曲目提供下载入口。
- 下载使用已存在的 Dio 与新增直接声明的 `path_provider`，临时文件成功后才重命名为最终文件；取消、失败、进度、重试和离线播放均有明确状态。
- 下载中的任务现可明确“暂停”或“取消”：暂停保留 `.part`，继续时携带 HTTP `Range` 并用 append 模式续传；服务器未返回 `206` 时丢弃已追加的无效临时文件并从头安全重试，避免把完整响应拼接成损坏文件。取消会删除临时文件。
- 恢复或应用重启后的未完成任务保留最终目标路径和临时文件状态，仍先显示“已暂停”并等待用户操作；这不是后台服务或自动续传承诺。
- 下载页已允许移除已完成或已取消任务；移除同时删除应用私有下载文件、临时文件和 SQLite 任务记录，文件已被系统清理时按幂等成功处理。

- 真机取链下载、关闭进程后的恢复、Android/iOS/鸿蒙后台调度和系统下载目录导出未验收；前台持久任务能力保持 `DOING`。
- 继续任务：重启恢复将核验已完成任务的目标文件；不存在时改为可重试失败，运行中/等待中仍转为用户可控暂停，遗留 `.part` 不删除以供 Range 续传。此轮不扩大为后台任务。
