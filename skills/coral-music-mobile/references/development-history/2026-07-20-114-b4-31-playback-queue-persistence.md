# B4-31 播放队列持久化

- 阶段：Batch 4 / Phase 3
- 状态：DONE
- 负责人：Codex
- 开始时间：2026-07-20
- 完成时间：2026-07-20

## 目标、范围与依赖

将当前播放队列、当前索引、播放模式和上下文保存到现有 SQLite；应用重启后恢复可见队列，但不自动播放或重新取在线地址。

复用 `LibraryStore` 的曲目快照序列化和既有队列控制器。不新增依赖，不持久化 User API 脚本、播放 URL、音量或随机历史。

## 实施记录

- 2026-07-20：开始实现。持久化将挂在共享队列状态变化处，避免为每个播放入口重复写库；异步恢复只能在用户尚未建立新队列时生效，防止启动竞态覆盖用户操作。
- 2026-07-20：`LibraryStore` schema 升级至 v9，新增单行 `playback_queue`；仅存曲目快照、索引、模式和上下文。无效快照自动删除，不影响启动。
- 2026-07-20：队列 provider 以串行写入保存状态；恢复不会自动播放，也不会覆盖启动期间已发生的用户队列操作。新增恢复索引钳位测试。
- 2026-07-20：复查新库创建路径时补上 v9 建表调用，确保全新安装与旧库升级都具备 `playback_queue`。
