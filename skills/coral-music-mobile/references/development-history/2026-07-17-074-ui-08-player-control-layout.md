# UI-08 播放详情主控布局

- 阶段：高保真 UI 重构 / 播放详情增量
- 状态：DONE
- 负责人：Codex
- 开始时间：2026-07-17
- 完成时间：2026-07-17

## 目标、范围与依赖

将播放详情的上一曲、播放/暂停、下一曲固定为视觉居中的主控制行；将随机/循环模式从该行移至单独的下一行，避免四个按钮导致主播放键偏移。依赖 UI-04、B4-03、B4-05，复用既有 `PlaybackQueueController.cycleMode`，不改变队列或模式业务语义。

## 桌面基线与实施方案

桌面端保留队列模式独立于前后切歌；移动端以更窄屏的主控优先级实现相同能力。修改仅在 `PlayerDetailPage`：主控行只保留上一首/播放/下一首，模式按钮改为居中的次级操作行，并维持语义 tooltip 和触控尺寸。

## 验收

播放中、暂停和队列长度为 1/多首时，主控行中心播放键不随模式按钮出现与否偏移；模式按钮仍可循环切换。完成后记录静态/Widget 与 Android 真机视觉验证。

## 实际修改与验证

- `lib/features/player/view/player_detail_page.dart`：主控行仅保留上一首、圆形播放/暂停、下一首；模式切换改为紧随其后的居中 `TextButton.icon`。两侧控制与 70px 主播放键保持对称间距。
- 自动验证：相关播放器回归 15 项通过，Dart 分析与 Android Debug 构建通过。
- 真机验收：Samsung SM-N986U / Android 13 真实播放详情截图确认上一曲/播放/下一曲位于同一居中主控行，列表循环按钮独立为下一行，窄屏下不再挤压主播放键。截图：`/private/tmp/coral-player-controls-sq.png`。
- 本任务状态更新为 `DONE`；仅改变表现层，不影响 B4-05 已验收的模式切换语义。
